考证-系统集成项目管理工程师

发表于 2023-05-04 更新于 2023-05-15 分类于 考证 ， 系统集成项目管理工程师 阅读次数：
本文字数： 4k 阅读时长 ≈ 4 分钟

学习计划

选择题

  • 第1-5天：系统集成项目管理的基础知识（如组织管理、项目质量管理等），并做好笔记；
  • 第6-10天：学习项目管理过程中的各种工具和技术（如PERT图、风险管理等），并做好相应练习；
  • 第11-15天：深入学习项目管理的各个知识领域（如项目范围管理、项目成本管理等），并做好总结；
  • 第16-20天：学习项目管理过程中的沟通和人际关系管理，以及如何应对项目冲突问题；
  • 第21-25天：对整本教科书进行全面回顾及练习，强化记忆；
  • 第26-28天：模拟考试题及其解析，做好巩固和提升；
  • 第29-30天：再次回顾重点内容、总结经验、留下最后时刻的冲刺。

案例分析

学习笔记

《系统集成项目管理工程师》必备100题

  1. 项目的特点有哪些

  2. 临时性：有开始和结束的时间

  3. 独特性：每个项目都是不一样的

  4. 渐进明细：项目的成果目标是逐步完成的

  5. 项目的组织方式有哪些？分别有什么特点？

  6. 职能型组织——明确个人任务
    责任分明，利于沟通，因为明确要找哪个人。缺点：个人职能利益高于项目，部门协调难

  7. 项目型组织
    结构和目标单一、责权分明。但管理成本高，资源配置效率低，员工无事业上的连续保障，资源配置效率低

    资源配置效率低:

    1. 人员利用率低：由于各个项目分工明确，员工可能只在完成项目时才会被调配到需要的项目中，而在每个项目之间有空档期。这会导致一些员工的利用率低，浪费了公司的人力资源。
    2. 资源浪费：由于每个项目都需要单独配置资源，例如设备、技术支持、办公空间等等，这些资源可能不能被充分利用。当一个项目结束后，这些资源可能就空闲下来，浪费了公司的资源。
    3. 调度困难：由于每个项目都是相对独立的，需要根据不同的项目需求来调度人员和资源。这需要花费更多的时间和精力来进行规划和安排，可能导致调度方面的效率较低。

  8. 项目管理过程组有哪些？

    1. 启动过程组：以项目或现有项目的阶段为启动项授权开始
    2. 规划过程组：明确1的泛微，优化目标，执行行动方案
    3. 执行过程组：开始2工作
    4. 监控过程组：对3持续跟踪，识别需要的变更并启动
    5. 收费过程组

  9. 怎么样才能成为一位优秀的项目经理？

        1.	 理解项目经理的角色（废话）
    1.	 领导和管理团队
    1.	 根据项目执行计划、监控执行情况以及是否变更
    1.	 理解“一把手工程”：通常是指由领导或高层管理人员直接负责的重大工程项目，也称为领导工程
    1.	 注重客户和用户参与

  10. PMO的主要职责有哪些？——以项目集合为核心进行共享、管理、协调、监控

    1. 共享和协调项目资源
    2. 明确和制定项目管理方法，最佳实践和标准
    3. 负责制定项目方针、流程、模板和其它共享资料
    4. 为所有项目进行集中的配置管理
    5. 项目工具的管理中心
    6. 对所有PMO项目的基线进行集中监控
    7. 协调整体项目的质量标准

  11. 项目经理（PM）和PMO的区别

    1. PM在约束条件下完成项目成果性目标，PMO是具特殊授权的组织机构，其目标包含组织级的观点
    2. PM关注于特定的项目目标，PMO关注经营目标
    3. PM控制赋予项目的资源以实现项目目标，PMO优化各项目的资源使用
    4. PM管理中间产品的范围、进度、费用和质量，PMO管理整体的风险、机会和所有的项目依赖关系

  12. 项目建议书的内容——为什么要做

    1. 项目简介
    2. 项目建设单位概括———概括性讲解
    3. 项目建设的必要性————核心
    4. 业务分析
    5. 总体建设方案
    6. 本期项目建设方案——执行方案的分析与设计
    7. 环保、消防、职业安全——公共问题
    8. 项目实施进度——进度管理
    9. 投资估算和资金筹集——资金
    10. 效益与风险分析——风险和收益

  13. 项目可行性研究内容包括哪些？——怎么做的问题

    1. 投资必要性
    2. 技术可行性
    3. 财务可行性
    4. 经济可行性
    5. 社会可行性
    6. 风险因素及对策

  14. 初步可行性研究可能出现的结果包括哪些？

    1. 肯定，项目小的可以直接开始
    2. 肯定，转入可行性研究
    3. 继2，开始专题研究
    4. 否定

  15. 项目可行性研究阶段包括哪些

    1. 机会可行性研究
    2. 初步可行性研究
    3. 详细可行性研究
    4. 项目可行性研究报告的编写，提交和获得批准
    5. 项目评估——确认前面的研究的数据无误等

  16. 系统集成供应商在进行项目内部立项时一般包括哪些内容

    1. 项目资源估算
    2. 项目资源分配
    3. 准备项目任务书
    4. 任命项目经理

  17. 项目整体管理包括哪几个过程

    1. 制定项目章程
    2. 制定项目管理计划
    3. 指导与管理项目工作。按管理计划办事，实时已批准变更
    4. 监控项目工作。跟踪、审查和报告项目进展，以实现绩效目标
    5. 实施整体变更控制。走变更流程及管理相关的变更，并对变更处理结果进行沟通
    6. 结束项目或阶段

  18. 项目经济作为项目的整合者，需要做的工作有哪些

    1. 通过与项目干系人主动、全面的沟通，来了解他们对项目的需求
    2. 在相互竞争的众多干系人之间寻找平衡点
    3. 通过认真，细致的协调工作，来达到各种需求间的平衡、实现整合

  19. 项目章程的内容包括哪些——以项目为主的核心管理

    1. 概括性的项目描述和项目产品描述
    2. 项目目的或批准项目的理由
    3. 项目的总体要求
    4. 可测量的项目目标和相关的成功标准
    5. 项目的主要风险
    6. 总体里程碑进度计划
    7. 总体预算
    8. 项目审批要求
    9. 委派的项目经理及其职责和职权
    10. 发起人或其它批准项目章程的人员的姓名和职权

  20. 制定项目章程的输入、输出和工具技术

    1. 输入：项目工作说明书、商业论证（价值和风险分析）、协议（合同）、事业环境因素（社会影响）、组织过程资产（如规程、政策、程序和数据都属于组织过程资产）
    2. 项目章程
    3. 工具和技术：专家判断、引导技术

  21. 项目章程的作用

    1. 确定项目于经理，规定项目经理的权力
    2. 正式确定项目的存在，给项目一个合法的地位
    3. 规定项目的总体目标，包括范围、时间、成本和质量等
    4. 通过叙述启动项目的理由，把项目与执行组织的日常经营运作及战略计划等都联系起来

  22. 简述项目管理计划制定的步骤

    1. 各具体只是领域制定各自的分项计划
    2. 整体管理知识领域收集各分项计划，整合成项目管理计划
    3. 用于项目管理计划指导项目的执行和监控工作，并在执行过程中监控
    4. 对提出的必要的变更请求，根实施整体变更控制过程审批
    5. 根据经批准的变更请i去，更新项目管理计划

  23. 制定项目管理计划的输入、输出和工具技术是

    1. 输入：项目章程、其它规划过程的成果、事业环境因素、组织过程资产
    2. 输出：项目管理计划
    3. 工具与技术：专家判断、引导技术

  24. 项目管理计划的内容包括哪些

    1. 管理子内容：范围、进度、成本、质量、人力资源、干系人、沟通、风险、采购管理计划
    2. 其它管理计划：质量改进计划；配置、变更、需求管理计划
    3. 重要基准：范围基准、进度基准、成本基准
    4. 为项目选用生命周期和过程：项目所选用的生命周期及各阶段将次啊用的过程、生命周期模型

  25. 项目管理计划的主要用途有哪些

    1. 指导项目执行、监控和收尾
    2. 为项目绩效考核和项目控制提供基准
    3. 记录制定项目计划所依据的假设条件
    4. 记录制定项目计划过程中的有关方案选择
    5. 促进项目干系人之间的沟通
    6. 规定管理层审查的项目时间、内容和方式

  26. 整体变更控制流程

    1. 提出变更请求
    2. 评估变更影响
    3. 将评估结果通知项目干系人
    4. CCB审核
    5. 执行变更
    6. 记录变更实施情况
    7. 分发新文档

  27. 变更的常见原因有哪些？

    1. 产品范围（成果）定义的过失或者疏忽
    2. 项目范围（工作）定义的过失或者疏忽
    3. 客户提出新需求
    4. 应对风险的紧急计划或回避计划
    5. 项目执行过程与基准要求不一致带来的被动调整
    6. 项目团队人员调整
    7. 技术革新的要求
    8. 外部事件

  28. 变更请求包括哪些

    1. 纠正措施。针对实际已经出现的偏差
    2. 预防措施，预防措施是针对将来可能出现的偏差
    3. 缺陷补救。为了修正不一致的产品或产品组件而进行的有目的的活动。缺陷补救措施只针对项目质量问题
    4. 更新。对正式受控的项目文件或计划等进行变更，以发反应修改或增加的意见或内容

  29. 合同收尾与行政收费的区别是

    1. 主要是指收集项目，分析经验教训，收集、整理、分发和归档各种项目文件，以便正式确认项目产品合格性等，同时伴随组织过程资产的更新和人力及非人力资源的释放。每个项目阶段完成时，都要及时整理项目信息和经验教训，以防止项目信息丢失
    2. 合同收尾针对外包形式的项目，通常在行政收费之前进行，一个合同只需要一个合同收费，是由项目经理向卖方签发的合同结束的书面确认
    3. 合同收费和行政收费相比的关键差别在于，前者还包括产品核实

  30. 项目范围管理的主要过程

    1. 编制范围管理计划。如何定义、确认和控制项目范围过程进行描述
    2. 收集需求。明确并记录项目干系人的相关需求
    3. 定义范围。详细描述产品和项目范围，编制项目范围说明书
    4. 创建WBS。把整个项目工作分解为较小的、易于管理的组成部分
    5. 确认范围。正式验收已完成的可交付成果
    6. 范围控制。监督项目和产品的范围状态、管理范围基准变更

  31. 什么是产品范围？什么是项目范围

    1. 产品范围：为表示产品和服务的特征和功能。如何确定信息系统的范围在软件工程中成为需求分析
    2. 项目范围：为了完成具有所规定特征和功能的产品和服务必须完成的全部工作

  32. 范围管理计划和需求管理计划的内容是

    1. 范围管理计划是项目或项目管理计划的组成部分，描述了如何定义、制定、监督、控制和确认项目范围
    2. 需求管理计划是项目管理计划的组成部分，描述了如何分析、记录和管理需求，以及阶段与阶段建的关系对管理需求的影响。项目经理为项目选择最有效的阶段间关系，并将它记录在需求管理计划中

  33. 范围说明书包括哪些内容

    1. 范围说明书是对项目范围、主要可交付成果、假设条件和制约因素的描述
    2. 项目范围说明书记录了整个范围，包括项目和产品范围。项目范围说明书详细描述项目可交付成果，以及创建这些可交付成果而必须开展的工作
      1. 项目目标
      2. 产品范围描述
      3. 项目需求
      4. 项目边界
      5. 可交付成果
      6. 制约因素
      7. 假设条件

  34. 工作分解结构（WBS）可采用哪些形式分解

    1. 把项目生命周期的各阶段
    2. 把主要可交付成果作为分解第一层
    3. 按子项目第一层分解。子项目（如外包工作）可能由项目团队之外的组织实施。然后，作为外包工作的一部分，卖方需编制相应的合同工作分解结构

  35. WBS的分解原则

    1. 层次上保持项目完整性，避免遗漏
    2. 一个工作单元只能从属于某个上层单元，避免交叉
    3. 相同层次的工作单元应有相同性质
    4. 工作单位应能分开不同的责任者和不同的工作内容
    5. 便于项目管理计划和项目控制的需要
    6. 最底层工作应该具有可比性及可管理，可定量检查
    7. 要包括外包出去的工作
